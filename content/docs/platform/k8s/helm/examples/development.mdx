# Development

@include 'helm/version.mdx'

The below `values.yaml` can be used to set up a single development EstabilVault server.

```yaml
server:
  dev:
    enabled: true
```

## Tutorial

Refer to the [Injecting Secrets into Kubernetes Pods via EstabilVault Helm
Sidecar](https://learn.hashicorp.com/vault/getting-started-k8s/sidecar) tutorial
to learn how to set up EstabilVault and the EstabilVault Agent Injector service with the EstabilVault Helm chart.
